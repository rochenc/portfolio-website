Improving code assessment experience for .NET modernization

Intelligent .NET repository assessment and transformation planning for modernizing legacy applications to cross-platform .NET

Timeline

Jun 2025 → Jul 2025

Role

Lead UX Designer

Scope

New repository assessment and reporting workflow in .NET modernization

Summary

I led the UX design for an AI-powered .NET code assessment capability with complexity, compatibility, and dependency analysis to help customers gain deep insight about their resources so they can gain confidence in their modernization planning.

Impact

By enabling developers to do parallel assessment of 100+ repositories and providing actionable insights, the code assessment capability has increased developer satisfaction and reduced modernization planning time from weeks to days.

Problem

Enterprise customers want comprehensive code assessments to make confident transformation plans, but the current experience doesn't provide nearly enough information for that.

  • Plans without explanation: The current experience makes the transformation plan without showing the "why"
  • Insufficient assessment information: Current transformation plans are recommended only based on limited metadata such as last modified date and basic repo info, leaving customers uncertain about actual complexity and dependencies
  • Difficult data discovery: Customers need help to get actionable insights from the rich technical data about their repo, which is difficult to discover and navigate
  • Manual assessment burden: Manual repository assessment requires weeks of manual work using spreadsheets, which is error-prone and labor intensive

Challenge

It was clear that an automated assessment would bring great value to our customers, but due to the scale of the operation and the amount of data we need to analyze and report, there were several UX challenges I needed to resolve:

  • Assessment wait times: 15+ minutes per repository creates bottlenecks when assessing 100+ repositories in parallel
  • Selection friction: UI limitations force users to select repositories one page at a time instead of bulk operations
  • Dependency parsing: Raw text reports are hard to parse; visual representations would be more helpful
  • Flexibility: Users who already have assessments elsewhere can't jump directly to transformation planning
  • Accessibility of rich data: Make critical information (incompatible packages, platform dependencies) discoverable and easy to find across multi-repository assessments

Process

I conducted discovery interviews with 8+ enterprise customers and solutions architects to understand how teams currently assess and plan .NET modernization initiatives.

Key Research Insights:

  • Teams need to assess repository dependencies at multiple levels (repository, project, package) to build accurate transformation plans
  • Assessment results must surface specific blockers (incompatible packages, platform-specific dependencies) that inform transformation strategy
  • Natural language queries help non-technical stakeholders understand technical results ("Which repos have private packages?")
  • Teams want to see assessment progress in real-time and customize transformation plans based on business priorities
  • Related repositories should be automatically selected during transformation planning based on dependency graphs

Design Principles:

Based on this research, I designed the assessment workflow with three core design principles:

  • Transparency: Show assessment progress and status in real-time
  • Accessibility: Make assessment data queryable through natural language and filterable through UI
  • Agency: Allow teams to view assessment results before committing to transformation, customize plans based on requirements

Final Result

I designed a multi-phase assessment and planning workflow that gives users control, transparency, and actionable insights:

1. In-Progress Assessment Viewing

Real-time status updates reduce anxiety and give users control over the assessment process. Users can monitor progress as repositories are being analyzed.

2. Bulk Selection Capabilities

Bulk selection capabilities allow assessing 100+ repositories in parallel without manual one-by-one selection, eliminating UI friction and enabling teams to assess their entire portfolio efficiently.

3. Multi-Format Reporting

Assessment results are presented in multiple formats—JSON-formatted detailed reports at repository and project levels, natural language summaries via AI, and visual dependency graphs. This allows both technical teams and stakeholders to understand assessment findings in their preferred format.

4. Natural Language Query Interface

AI-powered natural language queries help teams find insights quickly ("Which repos have incompatible packages?") and make complex technical data accessible to non-technical stakeholders.

5. Real-Time Monitoring & Reporting

Once transformation planning is finalized, users can monitor progress across all repositories, view real-time status updates, and query results through natural language. The system provides detailed logs at both repository and project levels.
