Process

üí° Key research insights

Users need guided, not fully automated, experiences: While users want efficiency, they don't want to lose control over critical decisions. This led to designing "human-in-the-loop" (HITL) interactions at key decision points.

Chat alone isn't enough: Users don't want to do everything in chat and expect it to be smarter. We balanced chat-based AI interactions with traditional UI for complex data visualization.

Final result

Designed an AI-assisted workflow that guides users through four key phases of SQL Server modernization, each optimized for specific user needs and technical requirements.

[Assess]

Assess

AI automatically analyzes SQL Server databases and .NET applications to identify complexity, dependencies, and modernization readiness. Users receive comprehensive assessment reports with clear recommendations for next steps.

Design decision: Applied progressive disclosure - chat provides high-level summaries while detailed dependency tables are available in structured UI panels. This prevents cognitive overload during the discovery phase.

[Wave plan]

Wave plan

Based on assessment results, the system generates prioritized migration waves that group databases and applications by complexity and dependencies. Users can adjust plans through chat or bulk operations.

Design decision: Implemented chat-first paradigm for quick plan adjustments ("move db2 to wave 3") while providing UI-based bulk editing for complex reorganization. This hybrid approach matches user mental models for different types of planning tasks.

[Schema conversion]

Schema conversion

Automated conversion of SQL Server schemas and stored procedures to PostgreSQL-compatible formats. Human-in-the-loop decision points handle complex mappings and validation before proceeding.

Design decision: Applied HITL pattern at critical decision points where AI confidence is low or business logic requires human judgment. Users review and approve schema mappings before irreversible transformations, maintaining control over technical decisions.

[Data migration]

Data migration

Streamlined data transfer from SQL Server to Aurora PostgreSQL with real-time progress tracking, validation checkpoints, and rollback capabilities for production-ready migrations.

Design decision: Combined all three patterns - chat for status updates and quick queries, structured UI for detailed progress monitoring, and HITL checkpoints for validation before final cutover. This ensures users maintain visibility and control during high-stakes data operations.

Impact & outcomes

Adoption success: 117 customers adopted SQL Server modernization with 167 active users and 205 workspaces created since GA launch.
