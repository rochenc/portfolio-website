Summary

I led UX for a 0→1 Agentic SQL Server modernization workflow within AWS Transform, launched at re:Invent 2025. The challenge was designing an efficient experience for enterprise customers to speed up their database migrations. As UX lead, I defined the workflow from assessment, wave planning to modernization execution in a chat driven experience.

My impact lies in making the agentic AI-driven experience useful and usable. This includes defining human-in-the-loop moments, crafting chat content design, and making new AI-human interaction clear and intuitive.

Impact

Since launching to AWS enterprise customers worldwide in December 2025, we've seen more than 46 customer-created jobs with a 20% job completion rate, reducing modernization time with the same scope from months to weeks. The solution has also eliminated SQL Server licensing fees and freed up significant developer time for higher-value work.

Challenge

How might we guide users through a SQL database modernization journey with clarity, confidence and control—interacting with multiple AI agents?

This challenge centers on three major areas:

1. Dealing with AI output hallucinations and inconsistencies
Ensuring agents provide reliable, accurate results when handling complex database analysis and transformation logic.

2. Showing agent rationale and transparency
Helping users understand why agents make recommendations and how they arrived at their conclusions.

3. Finetuning of agent-generated results
Allowing users to review, modify, and improve agent outputs before execution to maintain control and quality.

Key Research Insights

From interviewing with 10 migration and modernization practitioners, we learned that some of our assumptions are wrong.

1. Users need guided, not fully automated, experiences
While users want efficiency, they don't want to lose control over critical decisions. This led to designing "human-in-the-loop" (HITL) interactions at key decision points.

2. Chat alone isn't enough
Users don't want to do everything in chat and expect it to be smarter. We balanced chat-based AI interactions with traditional UI for complex data visualization.

Design Iterations

One key workflow iteration happens where customers set the target input for each wave. The decision lies in enabling full automation for target configuration, or a step-by-step approach.

Initial approach where users configure target databases and applications for each step within every wave, providing granular control but requiring more manual input.

Consolidated approach where target configuration happens as part of the wave planning process, reducing friction and centralizing decisions within the wave workflow.

Streamlined approach enabling full automation for target configuration, allowing users to apply wave-level settings across all databases and applications with a single action.

Final result

Designed an AI-assisted workflow that automatically analyzes SQL Server databases and .NET applications, converts schemas, migrates data, and refactors application code while maintaining human oversight at critical decision points.

Chat-first, UI-supported paradigm

Chat handles intent capture, explanations, and quick plan edits, while UI manages structured data, dependencies, bulk operations, and validation.

Progressive disclosure

Layered information architecture from chat summaries to detailed data tables to drill-down views.

Human-in-the-loop (HITL)

Clear decision points where users review and approve AI recommendations with full transparency.

Outcome

The SQL Server modernization solution was featured as a key announcement at AWS re:Invent 2025, highlighting the power of agentic AI to accelerate enterprise cloud migration workflows.
